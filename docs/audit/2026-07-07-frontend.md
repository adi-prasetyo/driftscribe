# DriftScribe frontend audit — 2026-07-07

Scope: `frontend/` (Svelte 5 + Vite SPA). Read-only audit. Findings carry file:line; a couple of the load-bearing claims were spot-checked against the code (noted inline). Severity is the auditor's; a follow-up reviewer should re-verify before acting.

## Findings (ranked)

### 1. (Medium) No stream cancellation anywhere — superseded `/chat` SSE runs keep executing
`frontend/src/App.svelte` `submitChat` (~430-646) bumps a `runSeq` guard that every `consumeSse` handler checks before writing state, but **nothing cancels the underlying fetch/reader**. `frontend/src/lib/sse.ts` `consumeSse` (line 181) takes no `AbortSignal`; `frontend/src/lib/api.ts` `apiFetch` (line 53) never passes one. **Verified:** `grep -rn "AbortController\|signal:" frontend/src/` returns zero hits in the whole app, and `consumeSse(resp, h)` has no signal parameter.

So `newChat()` (line 761), `openTrace()` (line 669), and `openConversation()` (line 317) all bump `runSeq` to "cancel" an in-flight run, but the old `fetch()`/`ReadableStream` reader keeps pumping to completion server-side. On the public demo (per-IP rate-limited by the CF Worker; 429 handling at App.svelte:513-518), a visitor who rapid-fires "new chat" mid-stream burns their own rate-limit budget on abandoned runs with no client signal to stop them.

**Fix:** thread an `AbortController` through `apiFetch`/`consumeSse`; abort the previous one at each of the three cancel points.

### 2. (Medium) `AuthPanel.svelte` (the operator-token modal) reimplements a dialog and is the least accessible one in the app
`Modal.svelte` (used elsewhere) is built on native `<dialog>` with a real focus trap, focus restore on close, and app/window-level Escape. `AuthPanel.svelte` instead is a plain `<div role="dialog" aria-modal="true">` (lines 57-64 — **verified**): no focus trap (Tab from the Save/Cancel buttons at 86-96 can leave the dialog), no focus restore on close, and Escape only closes via `onInputKeydown` (line 35) wired to the `<input>`'s own `onkeydown` — so Escape does nothing once focus moves to Save/Cancel. It also skips `lockBodyScroll`/`unlockBodyScroll` (`lib/scrollLock.ts`), so the page scrolls behind the backdrop. This is the one modal that can appear unprompted (any 401 triggers `requestToken()` at App.svelte:217).

**Fix:** rebuild it on `Modal.svelte`, or port the missing behaviors (focus trap + restore, window-level Escape, scroll lock).

### 3. (Low) Onboarding tour never moves focus when it opens
`App.startTour()` (App.svelte:203) flips `tourOpen=true`; the card's `$effect` (TourCard.svelte:45-53) only scrolls the spotlighted panel into view and toggles a CSS class — it never `.focus()`es the card or its close button. A keyboard/screen-reader user who clicks the header "Tour" button gets no signal anything happened. It's docked/non-modal by design so no full focus trap is warranted, but an initial focus move is a cheap missing piece (compare the deliberate focus moves in `openTrace`/`openConversation` to `#historical-badge`/`#conversation-thread`).

## Verified NOT a defect (explicitly in scope)
- **Mermaid is genuinely lazy/code-split.** `npm run build`: main SPA bundle `transparency-*.js` is 209KB / 72KB gzip; `mermaid` (~620KB across `mermaid.core` + diagram chunks) only enters via `await import('mermaid')` at `InfraDiagram.svelte:305`, gated on `previewActive` (PR-preview ghost). No accidental eager bundling.

## Done well (don't touch)
- **URL/href hygiene is exemplary and consistent** wherever an href touches server/model-authored data: `lib/approval.ts` (`safeApprovalHref`/`safeGithubHref`/`iacApprovalHref` — scheme+host+path allowlist, same-origin checks, userinfo rejection) and `lib/markdown.ts` (`safeMarkdownLinkHref`, same rigor for the hand-rolled Markdown subset that renders agent PR bodies in `PrBodyDisclosure.svelte`). Every anchor traced (`ConversationThread.svelte:93`, `DecisionsRail.svelte:174/250/259/269`, `InfraDiagram.svelte:733`) routes through one of these guards. The only two `{@html}` uses are justified: `Icon.svelte` (static hardcoded icon paths, no external data) and `InfraDiagram.svelte:665` (Mermaid SVG rendered with `securityLevel:'strict'` + `htmlLabels:false`, and every dynamic label pre-escaped in `lib/infra_graph.ts:escapeMermaidLabel:240`) — real defense in depth.
- **Timer/listener lifecycle discipline is tight.** `lib/infra_refresh.ts` (`RefreshScheduler`) and the header popovers (`AutonomyPill.svelte`, `PausePill.svelte`) pair every `setTimeout`/`setInterval`/`ResizeObserver`/`addEventListener` with cleanup, guarded by monotonic epochs so a stale async callback can't clobber newer state. `lib/scrollLock.ts` is ref-counted for overlapping modals. No leak found.
- **`lib/timeline.ts` `reconcileBackfill`** (live-stream vs `/trace` snapshot merge) is sound; its one limitation (can't recover a non-mcp tail if the stream dies mid-turn) is honestly documented in the comment, not swept under.

## Not exhaustively traced
`ConversationsRail`/`DecisionsRail` search-modal keyboard nav and the `pauseStore`/`autonomyStore` race-guard logic were sampled (looked consistent with the well-built pattern) but not exhaustively branch-traced.

DriftScribe — Architecture

Coordinator + four execute-only workers on Cloud Run. Layer-0 tool registry, per-worker hardcoded policies, HITL gate on destructive ops. See docs/architecture/multi-agent-design.md for the full prose.

1. Trigger fan-in

Three trigger sources fan in to the Coordinator, which delegates to four workers via audience-bound ID tokens. The HITL diamond gates the Coordinator→Rollback edge; rollback /execute only fires after an operator approves at /approvals/{id}.

2. Layered safety boundary

For each service, an outer dashed box (Layer 1: IAM scope of its deploy-time service account) wraps an inner solid box (Layer 2: hardcoded payload-intent policy). The Coordinator's inner box is intentionally empty — it has no mutate caps on payment-demo. The HITL diamond intercepts the Coordinator→Rollback flow.

Reading the rows: Layer 1 (left) is what the service account could do at the Google Cloud IAM layer; Layer 2 (right) is what the worker code will do regardless of request shape. Both must be breached to mutate payment-demo, and the Rollback path additionally requires a one-shot HMAC token minted by operator action.
